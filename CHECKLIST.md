QA Check-list Specification

Назначение документа

Данный документ описывает чек-лист для ручного тестирования веб-сайта заказчика. Чек-лист используется для первичной и регрессионной проверки удобства, функциональности и корректности вёрстки сайта.

Инструкция по использованию
	1.	Откройте сайт заказчика.
	2.	Скопируйте данный чек-лист в рабочий документ.
	3.	Пройдитесь по всем пунктам чек-листа.
	4.	Некоторые пункты необходимо проверять на всех страницах сайта.
	5.	Пункты, связанные с полями ввода, применяются к каждому полю.
	6.	При обнаружении ошибки:
	•	зафиксируйте дефект;
	•	сделайте скриншот;
	•	приложите скриншот к чек-листу;
	•	опишите ожидаемый и фактический результат.

Чек-лист: Удобство сайта
	•	Проверка правильности содержания на веб-страницах (отсутствие грамматических и орфографических ошибок).
	•	Выравнивание текста единообразно, элементы выглядят ровно и эстетично.
	•	Сообщения об ошибках не содержат орфографических ошибок и понятны пользователю.
	•	У всех полей есть подсказки и отображается корректный формат заполнения.
	•	Между элементами интерфейса (поля, кнопки и т.д.) присутствует достаточное пространство.
	•	Элементы дизайна не наслаиваются друг на друга.
	•	Все кнопки выполнены в едином стиле и стандартного размера.
	•	На всех страницах присутствует ссылка на домашнюю страницу.
	•	Недоступные для заполнения поля отображаются серым цветом.
	•	Отображаются подтверждающие сообщения об успешной отправке формы или добавлении данных.
	•	Система работает стабильно, без торможений и затруднений при выполнении операций.
	•	Элемент скроллинга отображается только при необходимости.
	•	На каждой странице присутствует заголовок.
	•	Все поля (текстовые, выпадающие списки, радио-кнопки и т.д.) и кнопки доступны с клавиатуры.
	•	Данные в выпадающих списках не обрезаются и не перекрываются другими элементами.

⸻

Чек-лист: Общее
	•	Сайт корректно открывается и доступен.
	•	Повторное открытие сайта выполняется без сбоев.
	•	Все кнопки на сайте реагируют на нажатие.
	•	Все ссылки переходят на соответствующие страницы.
	•	Отсутствуют битые ссылки.
	•	Выполняется проверка всех веб-форм на сайте.
	•	Обязательные поля успешно проходят валидацию.
	•	Обязательные поля помечены символом «*».
	•	Необязательные поля также проходят валидацию.
	•	Основные элементы сайта функционируют без нарушений.
	•	Навигационное меню работает корректно.
	•	Загружаемые документы открываются корректно.
	•	Отправка форм происходит без ошибок.
	•	Проверено поведение сайта при удалении cookies.
	•	Проверена реакция сайта после повторного посещения при удалённых cookies.
	•	Все данные в списках отображаются в хронологическом порядке.

⸻

Чек-лист: Вёрстка
	•	Используется кодировка UTF-8.
	•	Шрифты успешно загружаются и корректно отображаются.
	•	Ошибки HTML и CSS отсутствуют.
	•	Элементы веб-страниц корректно отображаются на разных разрешениях экрана.
	•	Функциональность кнопок подтверждена на различных страницах.
	•	Вёрстка форм корректно адаптируется при изменении размеров окна.
	•	Сайт корректно отображается в разных веб-браузерах.
	•	В коде отсутствует избыточное количество комментариев.
	•	Сайт имеет favicon.
	•	В консоли браузера отсутствуют ошибки JavaScript.

Примечания
	•	Чек-лист является базовым и может быть расширен в зависимости от специфики проекта.
	•	Рекомендуется фиксировать результаты проверки для последующего анализа и регрессии.
